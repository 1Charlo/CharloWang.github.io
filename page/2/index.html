Linux中python虚拟环境的创建

发表于 2017-08-10 | 更新于 2018-11-19

Linux中一般都安装了python环境，通常为python2.7和python3.4，但在开发中我们通常要指定一个python的版本，同时也可能需要安装新的python版本，多个版本的混杂，当我们真正需要使用时就变得很麻烦。而虚拟环境virtualenv用于创建独立的Python环境，多个Python相互独立，互不影响可以用于创建独立的Python环境，多个Python相互独立，互不影响。

首先需要安装virtualenv：
sudo apt-get install python-virtualenv

然后指定python版本并安装到某个目录：
virtualenv --python=/usr/bin/python3.4 ~/Python3.4

如果想要不依赖系统中已安装好的第三方package，可以使用下面的命令：
virtualenv --no-site-packages --python=/usr/bin/python3.4 ~/Python3.4

这样做了后，一个虚拟环境就创建成功了。

JSONP(JSON with Padding)是JSON的一种“使用模式”，可用于解决主流浏览器的跨域数据访问的问题。由于同源策略，一般来说位于 server1.example.com 的网页无法与不是 server1.example.com的服务器沟通，不管是静态页面、动态网页、web服务、WCF，只要是跨域请
求，都是无法通过的，而HTML的<script>元素是一个例外。利用<script>元素的这个开放策略，网页可以得到从其他来源动态产生的JSON 资
料，而这种使用模式就是所谓的 JSONP。其实不仅是<script>标签，凡是拥有src属性的标签，都具有跨域的能力。
　　JSON是一种数据交换格式，而JSONP是一种依靠开发人员的聪明才智创造出的一种非官方跨域数据交互协议。一个是描述信息的格式，一个是信息传递双方约定的方法。

安装git

git安装完成后，生成SSH密钥：

ssh-keygen -t rsa -C “你的邮箱地址”，然后一直回车。
去用户目录下的.ssh文件夹中，找到两个文件id_rsa和id_rsa.pub。
打开id_rsa.pub，复制全部内容，打开https://github.com/settings/ssh ，Add SSH key，将刚复制的内容粘贴进去。
